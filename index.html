What are the new features introduced in HTML5?

  • Semantics: New semantic elements like'header','footer',"article", "section", nav and aside were introduced o better define the structure of web pages, making them more readable for both humans and machines
  • Audio and video Support: Native support for embedding audio and video without third party plugins, using the audio and video element.
  • canvas: The canvas element allows dynamic, scriptable rendering of 2d shapes and bitmap images.
  • local storage: The localStorage and sesssionStorage APIs provide stoarge solution within the browser
  • Geolacation HTML5 introduced the geolocation api, enabling website to access a users geographical locatiton throught the browser
  • WebSockets: Facilitates real time communication between a client and a server enabling bidirection communication.
  • Responsive Design: HTML5 encourages responsive web design by providing better support for media queries, enabling websites to adapt various screen sizes and devices
  • Form Enhancements: New input types like 'email','url','date','number', ets.. and attributes such as 'placeholder','required', and 'pattern' were introduced to improve form handling and validation.

Explain the basic structure of an HTML document.

  • Document Type Declaration (DOCTYPE) This is an instruction to the web browser about the version of HTML the page is written in. it is declared at the very beginning of the HTML document.
  • HTML element The html element is the root element of an html page. it encapsulates the entire content of the page
  • Head Section The head section contains meta-information about the html document such as the title of the page, links to external resources like css stylesheets or scripts and meta tags

Explain the difference between inline and block elements.

  • Block Elements: Display: block-level elements typically start on a new line and occupy the full widgh available. Examples include div p h1 - h6 ul ol li table etc Width and Height: Block elements by default, take up the full width available and expand to fill the entire horizontal space unless a specific with is defined. They also respect height attributes and margins, which contribute to pushing subsequent elements down. Structure: These elements are generally used for larger structures and to create the main divisions or sections of a web page.
  • Inline Elements: Displaylnline elements do not start on a new line. They typically occupy only as much width as necessary, allowing other elements to sit beside them. Examples include span a strong em img input b i tags etc Width and Height: Inline elements do not respect width and height attributes. They only occupy the space required by their content. Usage:lnline elements are used within block-level elements to style specific parts of the content, create hyperlinks, emphasize text, or insert images within text.
  • Block elements usually start on a new line, take up the full width available, and allow setting width and height. Inline elements do not start on new lines, occupy only necessary width, and do not allow setting width and height attribute
